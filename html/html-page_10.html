Атрибуты тега <form>

Формы задаются при помощи тега <form> , который является парным. Содержимое формы на 99% состоит из тегов <input> различного типа. Этот тег является одиночным.

Пример простой формы

<form action="main.php" accept-charset="utf-8" method="post" name="test" enctype="application/x-www-form-urlencoded" autocomplete="off" novalidate >  
  <input name="name"> 
  <input name="mail"> 
  <input name="text"> 
</form>

Атрибуты тега <form>

  • action Указывет путь к файлу - обработчику
  • accept-charset Устанавливает кодировку для отправляемых данных. Данные, отправляемые формой имеют ту же кодировку, что и страница, на которой находится форма. Но как же быть, если страница например в кодировке UTF-8, а надо чтобы данные отправлялись в кодировке windows-1251. Конечно есть такой вариант, как сменить кодировку у страницы. Но это не всегда возможно, сайт — CMS уже настроена на работу с конкретной кодировкой. Поменять кодировку отправляемых формой данных можно прописав у тега form атрибут accept-charset=»нужная кодировка».
  • method Указывает как передовать данные:
    • get ( по умолчанию ) ограничивает объём передаваемой скрипту информации, максимальный объем здесь 4 Кб. Так как метод GET отправляет скрипту всю собранную информацию формы как часть URL (то есть в открытом виде), то это может пагубно повлиять на безопасность сайта;
    • post Метод POST отличается от GET тем, что данные передаются в закрытой форме. Объем передаваемых значений методом POST по умолчанию ограничен и равен 8 Мбайт
  • name Задается уникальное имя формы, используется в основном JavaScript. Имена могут состоять только из латинских букв, цифр и дефисов. MyForm или SamajaLutchaja-Forma_125
  • enctype Указывает способ кодировки отправленных данных.

    HTML-формы предоставляют три метода кодирования:

    • application/x-www-form-urlencoded по умолчанию
    • multipart/form-data
    • text/plain
    Специфика форматов не важна для большинства разработчиков. По этому используем "по умолчанию"
  • autocomplete Включает/выключает (on/off) автозаполнение форм. Значение может быть перекрыто атрибутом autocomplete у конкретных элементов формы.
  • novalidate Отменяет проверку на правильность заполнения полей форм

Тег lebel

Элемент <label> устанавливает связь между определённой меткой, в качестве которой обычно выступает текст, и элементом формы (<input>, <select>, <textarea>). Такая связь необходима, чтобы изменять значения элементов формы при щелчке курсором мыши на текст. Кроме того, с помощью <label> можно устанавливать горячие клавиши на клавиатуре и переходить на активный элемент подобно ссылкам.

Существует два способа связывания объекта и метки. Первый заключается в использовании идентификатора id внутри элемента формы и указании его имени в качестве атрибута for элемента <label>. При втором способе элемент формы помещается внутрь контейнера <label>. 

Вариант 1
<form action="#" >
  <label for="iden_1">Ваше имя:</label>
  <input id="iden_1" type="text" placeholder="Введите Ваше имя">
</form>

Вариант 2
<form action="#" >
  <label> Ваше имя:< input type="text" placeholder="Введите Ваше имя"></label>
</form>


С радио-кнопками и чек-боксами существует особенность. Идентификатор на то и идентификатор. что может присваивается только одному из элементов, но тогда в радио-кнопках становиться возможным выделение всех элеменов. чтобы этого не происходило всем элементы списка присваивается одно имя. При клике по тексту отмечается тот элемент списка у которого идентификатор. Возможно так же выделение одного из списка с помощью атрибута checked

Списки в формах

<select> позволяет создать элемент интерфейса в виде раскрывающегося списка, а также список с одним или множественным выбором. Каждый пункт создаётся с помощью элемента <option> , который должен быть вложен в контейнер <select>. Общий вид: 

<select>
  <option>Пункт 1</option>
  <option>Пункт 2</option>
</select>

Атрибуты <select>

  • autofocus ⟩⟩⟩⟩ Устанавливает, что список получает фокус после загрузки страницы, при этом список становится доступным для выбора пунктов, например, с помощью клавиатуры
  • disabled ⟩⟩⟩⟩ Блокирует доступ и изменение элементов списка. Блокированный список не может получить фокус через курсор или клавиатуру, быть изменён, значение такого списка не передаётся на сервер. Непонятно, зачем этот список делать, если он все равно будет заблокирован
  • form ⟩⟩⟩⟩ Связывает список с формой по её идентификатору. Такая связь необходима в случае, когда список располагается за пределами <form>.
  • multiple ⟩⟩⟩⟩ Наличие атрибута multiple сообщает браузеру отображать полное содержимое элемента <select> как список множественного выбора. Для выбора нескольких значений списка применяются клавиши Ctrl и Shift совместно с курсором мыши.
  • name ⟩⟩⟩⟩ Имя элемента для отправки на сервер или обращения через скрипты.
  • required ⟩⟩⟩⟩ Устанавливает список обязательным для выбора перед отправкой формы на сервер
  • size ⟩⟩⟩⟩ Количество отображаемых строк списка.

Атрибуты <option>

  • disablet ⟩⟩⟩⟩ Заблокировать для доступа элемент списка.
  • label ⟩⟩⟩⟩ Атрибут предназначен для указания метки пункта списка, сокращённой по сравнению с текстом внутри <option>. Если атрибут label присутствует, то текст внутри <option> игнорируется и в списке выводится значение label.
  • selected ⟩⟩⟩⟩ Заранее устанавливает определённый пункт списка выделенным.
  • value ⟩⟩⟩⟩ Значение пункта списка, которое будет отправлено на сервер или прочитано с помощью скриптов.

Атрибуты <optgroup>

Элемент <optgroup> представляет собой контейнер, внутри которого располагаются элементы <option>, объединённые в одну группу. Особенностью <optgroup> является то, что он не выделяется как обычный элемент списка, акцентируется с помощью жирного начертания, а все элементы, входящие в этот контейнер, смещаются вправо от своего исходного положения.

  • disabled Блокирует доступ к группе списка.
  • label ⟩⟩⟩⟩ Текст, который будет входить в список в виде заголовка группы. ОБЯЗАТЕЛЬНЫЙ!

Тег textarea

Представляет собой элемент формы для создания области, в которую можно вводить несколько строк текста. В отличие от элемента <input> в текстовом поле допустимо делать переносы строк, они сохраняются при отправке данных <textarea></textarea> сервер.

Между тегами <textarea> и </textarea> можно поместить любой текст, который будет отображаться внутри поля.

Атрибуты <textarea>

  • autocomplete Включает или отключает автозаполнение.
  • autofocus Автоматическое получение фокуса
  • cols ⟩⟩⟩⟩ Ширина поля в символах

  • disabled ⟩⟩⟩⟩ Блокирует доступ и изменение элемента
  • form ⟩⟩⟩⟩ Связывает текстовое поле с формой по ее идентификатору
  • maxlength ⟩⟩⟩⟩ Максимальное число введенный символов
  • minlength ⟩⟩⟩⟩ Минимальное число введенный символов
  • name ⟩⟩⟩⟩ Имя поля, предназначено для того, чтобы обработчик формы мог его идентифицировать.
  • plaseholder ⟩⟩⟩⟩ Добавляет подказку
  • readonly ⟩⟩⟩⟩ Устанавливает, что поле не может быть изменeно пользователем
  • required ⟩⟩⟩⟩ Обязательное к заполнению поле
  • rows ⟩⟩⟩⟩ Высота поля в строках текста
  • wrap ⟩⟩⟩⟩ Параметры переноса строк
    • soft Слова в текстовом поле переносятся механически, если они не поместились в размер области, но при отправке на сервер переносы не сохраняются.
    • hard Длинный текст, который самостоятельно не помещается в поле по ширине, автоматически переносится на новую строку и вместе с переносами отправляется на сервер. Для работы требуется наличие атрибута cols.

Тег fieldset

Элемент <fieldset> предназначен для группирования элементов формы. Такая группировка облегчает работу с формами, содержащими большое число данных, например, один блок может быть предназначен для ввода текстовой информации, а другой — для флажков.

Браузеры для повышения наглядности отображают результат использования элемента <fieldset> в виде рамки. Её вид зависит от операционной системы, а также используемого браузер

Атрибуты тега <fieldset>

  • disabled Блокирует поля формы в группе.
  • form Связывает группу с формой.

Тег legend

Тег <legend> ⟩⟩⟩⟩

Элемент <legend> применяется для создания заголовка группы элементов формы, которая определяется с помощью <fieldset>. Группа элементов обозначается в браузере с помощью рамки, а текст, который располагается внутри контейнера <legend>, встраивается в эту рамку.

Атрибут tabindex

Атрибут tabindex

Атрибут tabindex устанавливает порядок получения фокуса при переходе между элементами с помощью клавиши Tab. Переход происходит от меньшего значения к большему, например от 1 к 2, затем к 3 и так далее. При этом строгая последовательность не важна, допускается пропускать какие-то числа и начинать с любой цифры. Если значения tabindex у элементов совпадают, тогда учитывается их порядок появления в коде. Заблокированные элементы (у которых установлен атрибут disabled) не участвуют в переходе и фокус не получают.

Переход к элементам, у которых не задан атрибут tabindex или его значение равно 0, происходит после всех «нумерованных» элементов в том порядке, как они указаны в коде.

Для <input type="radio"> находящихся в одной группе tabindex одинаков.

Примеры type = "range"

Первоначальное положение ползунка это (min+max)/2 т.е. - середина шкалы. value="100"изменяют положение ползунка на указанное. Тег <output> выводит указанное в нем значение справа от шкалы. Для этого тега указываем идентификатор, чтобы привязать к нему скрипты. В данном случае для тега <input> следующее: oninput="out.value=(ran.value)" 

<form action="#">
  <label for="ran">Укажите Ваш рост (cm): <br>
  <input oninput="out.value=(ran.value)" id="ran" type="range" min="0" max="300" value="100"></label>
  <output id="out">100</output>
</form>
100

<form action="#">

  <p>Ваш рост <output id="out">100 </output> сантиметров</p>
  <p>или <output id="out2">1000</output> миллиметров</p>
</form>
