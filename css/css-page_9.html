Анимация

Анимация CSS3 поддерживает покадровую анимацию, которая позволяет создавать сложные анимации объекта.

Достигается она при помощи свойства animation и правила @keyframen

Важно: При помощи свойства мы вызываем анимацию, а в правиле мы его задаем и описываем как оно должно работать.

У animation 2 обязательных параметра - имя анимации (любое, хоть ёклмн, но на латинице) и ее продолжительность.

Мы можем задавать анимацию @keyframen двумя способами.

Первый это простая анимация from/to т.е от/до

<div class="animate"> Пример animation</div>
Пример animation
.animate {
  background: #ccc;
  border: 1px solid #bbb;
  padding: 5px 20px;
  width: 200px;
  margin: 0 auto;
  text-align: center;
  color: #444;
}
.animate:hover {
  animation:garmoshka 2s;
}
@keyframes garmoshka{
  from {width: 200px;}
  to {width: 400px;
    border-radius: 20px;
    transform:rotate(3deg);
  }
}

Второй - покадровая анимация, которую мы задаем при помощи процентов 0-100%

<div class="animate2"> 0% animation 100% </div>
0% animation 100% 
.animate2 {
  background: #ccc;
  border: 1px solid #bbb;
  padding: 5px 20px;
  width: 200px;
  margin: 0 auto;
  text-align: center;
  color: #444;
}
.animate2:hover {
  
  animation:dance 20s;
}
@keyframes dance{
  0% {}
  25% {
    width: 500px;
    border-right: 20px;
  }
  50% {
    width: 200px;
  }
  75% {
    border-right: 0;
  }
  100% {
    width: 300px;
    border-radius: 20px;
    transform:rotate(3deg);
  }
}

Несколько советов

1. Чтобы в конце анимации не было прыжка делайте последний слайд 100% такой же, как и оригинальное состояние тега

2. А чтобы вернуть оригинальное состояние - можно просто указать пустые фигурные скобки 100% {} .

3. Можно удалять одинаковые свойства, которые находятся в промужеточных кадрах.

4. Анимация не должна двигать другие элементы сайта. Поэтому готовте для нее плацдарм.

5. Если вы хотите двигать объекты - используйте позиционирование.

6. Если вы используете позиционирование, все свойства, которые вы будите указывать должны уже быть заданы вашему блоку.

Анатомия анимации

Свойство animation имеет много различных настроек для анимации объектов.

А именно:

  • animation-name - задает имя для анимации и является обязательным атрибутом.
  • animation-duration - задает время анимации и тоже является обязательным атрибутом.
  • animation-timing-function - кривая Безье, которая задает стиль анимации.
  • animation-delay - задает время задержки анимации.
  • animation-iteration-count - счетчик, задает количество повторов анимации (используем целые числа. Если хотим чтобы анимация была "безконечной", вместо числа ставим - infinite).
  • animation-direction - указывает как должна выполняться анимация
    • normal - анимация работает от начала и до конца ("по умолчанию").
    • reverse - анимация работает от конца в начало.
    • alternate - анимация работает от начала до конца, и обратно в начало.
    • alternate-reverse - анимация работает от конца до начала, и обратно в конец.
    • Важно: alternate и alternate-reverse работают только тогда, когда количество повторов анимации больше 1 (одного)!
  • animation-fill-mode - определяет как будет вести себя элемент до и после анимации.
    • none - после анимации элемент "возвращается на круги своя".
    • forwards - после анимации элемент "застывает" в крайнем кадре.
    • backwards - анимация начинает работать до начала анимации.
    • both - начинает работать сразу и "застывает" в крайнем кадре.
  • animation-play-state - отвечает за паузу.
    • running - анимация работает
    • paused - пауза

Важно

1. Чтобы анимация правильно работала необходимо соблюдать последовательность применения атрибутов. Если какой-то атрибут не нужен, просто пропускаем его.

2. Помним об "обязательных" атрибутах: имя и время (name and duration).
